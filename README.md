## 🎯 Complete Syllabus

### 1. Learning Basics

- input and output
- variables and data types
- operators
- conditions
- loops
- functions
- arrays and strings
- basic debugging

### 2. DSA Theory and Core Terms

- data, data type, abstract data type
- problem, constraint, test case, edge case
- brute force, optimal solution, trade-off
- recursion, iteration, state, transition
- time complexity and space complexity
- best, average, and worst case
- asymptotic notation: Big-O, Theta, Omega

### 3. Pattern Building with Loops and Conditions

- half pyramid
- inverted pyramid
- number patterns
- hollow patterns
- character patterns
- nested loops
- condition-based printing

### 4. Data Structures

- arrays
- strings
- linked lists
- stack
- queue
- deque
- hash table / hashing
- set and map
- heap / priority queue
- tree
- binary search tree
- trie
- graph
- disjoint set union

### 5. Algorithms

- linear search
- binary search
- sorting: bubble, selection, insertion, merge, quick, heap
- prefix sum
- two pointers
- sliding window
- recursion
- backtracking
- greedy
- divide and conquer
- dynamic programming
- graph algorithms

### 6. Advanced Topics

- bit manipulation
- segment tree
- fenwick tree
- shortest path
- minimum spanning tree
- topological sort
- monotonic stack and queue
- advanced DP states

## 💻 Language Tracks {#-language-tracks}

Each language folder is organized into:

- `01. Basics`
- `02. Patterns`
- `03. Data Structures`
- `04. Algorithms`

Recommended order:

1. Learn theory from the `Theory/` folder.
2. Practice patterns and basic logic.
3. Implement the same concept in all three languages.
4. Solve topic-based problems.
5. Track mistakes and revise weak areas.

### 🛠️ Setting Up MkDocs Locally

#### Prerequisites

- **Python 3.7+** installed on your system
- **pip** package manager

#### Installation Steps

1. **Install MkDocs and Material Theme**
   ```bash
   pip install mkdocs
   pip install mkdocs-material
   pip install mkdocs-mermaid2-plugin
   pip install mkdocs-glightbox
   ```

2. **Navigate to the docs directory**
   ```bash
   cd docs
   ```

3. **Start the local development server**
   ```bash
   mkdocs serve
   ```

4. **Open your browser**
   - Navigate to `http://127.0.0.1:8000`
   - View the interactive documentation

#### Building the Documentation

```bash
# Build static files
mkdocs build

# Deploy to GitHub Pages
mkdocs gh-deploy
```

## 💻 Local Setup

### Prerequisites

- **C++**: g++ compiler
- **Java**: JDK 8 or higher
- **Python**: Python 3.6 or higher
- **Git**: For version control

### Installation Steps

1. **Clone the repository**
   ```bash
   git clone https://github.com/Codes-RJ/DSA.git
   cd DSA
   ```

2. **Verify language installations**
   ```bash
   # Check C++ compiler
   g++ --version
   
   # Check Java
   java -version
   
   # Check Python
   python --version
   ```

3. **Navigate to your preferred language folder**
   ```bash
   cd "1. C++"    # or "2. Java" or "3. Python"
   ```

### Running Code

#### C++
```bash
g++ file.cpp -o run
./run
```

#### Java
```bash
javac FileName.java
```

#### Python
```bash
python file.py
```

## 🤝 Contributing

We welcome contributions! Here's how you can help:

### Contribution Guidelines

1. **Fork the repository** (see instructions above)
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow the existing code style and structure
   - Add comments for complex algorithms
   - Include time/space complexity analysis
   - Test your code thoroughly

4. **Commit your changes**
   ```bash
   git commit -m "Add: Brief description of your changes"
   ```

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Go to your fork on GitHub
   - Click "New Pull Request"
   - Provide a clear description of your changes

### What to Contribute

- **New algorithms** and data structures
- **Optimized solutions** for existing problems
- **Additional language implementations**
- **Bug fixes** and error corrections
- **Documentation improvements**
- **Practice problems** with solutions
- **Templates** for competitive programming

### Code Style Guidelines

- Use meaningful variable and function names
- Add proper comments for complex logic
- Include time and space complexity in comments
- Follow language-specific conventions
- Test with multiple test cases
