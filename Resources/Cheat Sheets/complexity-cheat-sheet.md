# Complexity Cheat Sheet

## Common Time Complexities

- array access: `O(1)`
- linear search: `O(n)`
- binary search: `O(log n)`
- hash map average insert/search: `O(1)`
- BFS and DFS: `O(V + E)`
- merge sort: `O(n log n)`
- quick sort average: `O(n log n)`
- simple nested loops over `n`: `O(n^2)`

## Common Space Ideas

- iterative loops usually need less stack space than recursion
- hash-based optimization often trades memory for speed
- in-place algorithms try to reduce auxiliary space
