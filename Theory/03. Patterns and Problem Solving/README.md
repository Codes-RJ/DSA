# Patterns and Problem Solving

Pattern recognition is the bridge between theory and problem solving.

## Pattern Printing with Loops

These train nested loops, conditions, and index handling.

- star pyramid
- inverted pyramid
- right triangle
- number triangle
- hollow rectangle
- character pattern

## Common DSA Patterns

- two pointers
- sliding window
- fast and slow pointers
- prefix sum
- binary search on answer
- recursion and backtracking
- greedy choice
- dynamic programming state transition

## Problem-Solving Process

1. understand input, output, and constraints
2. identify the likely pattern
3. write the brute force version
4. optimize
5. test edge cases

## Important Questions to Ask

- is the array sorted?
- do I need frequency counting?
- do I need a stack for previous/next greater logic?
- can I reuse previous results?
- do I need a graph traversal?

## Common Mistakes

- jumping to a complex method too early
- ignoring corner cases
- mixing up indices
- not checking complexity against constraints
