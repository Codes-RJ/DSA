# DSA Glossary

## Core Terms

- abstract data type: behavior-focused model such as stack or queue
- auxiliary space: extra memory beyond the input
- brute force: direct but usually slower approach
- dry run: manually tracing the program
- edge case: boundary or unusual input
- invariant: condition that remains true during execution
- in-place: uses very little extra memory
- stable sort: equal elements keep relative order
- traversal: visiting all elements or nodes systematically
- overflow: result exceeds allowed numeric range

## Pattern Terms

- state: information needed to describe a subproblem
- transition: rule that moves from one state to another
- window: continuous subarray or substring range
- pointer: index or reference used to move through data
