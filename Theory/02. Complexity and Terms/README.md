# Complexity and Terms

This section builds the language of DSA.

## Important Terms

- data structure: a way to organize data
- algorithm: a step-by-step solution process
- input size: number of items given to the program
- constraint: problem limit such as `n <= 10^5`
- edge case: unusual or boundary input
- brute force: straightforward baseline solution
- optimal: best solution under given constraints

## Complexity Analysis

### Time Complexity

How the running time grows as input size grows.

- `O(1)` constant
- `O(log n)` logarithmic
- `O(n)` linear
- `O(n log n)` common efficient sorting complexity
- `O(n^2)` nested loop behavior

### Space Complexity

How much extra memory the algorithm uses.

- input space
- auxiliary space
- recursion stack

## Asymptotic Notation

- Big-O: upper bound
- Theta: tight bound
- Omega: lower bound

## Common Interview Terms

- in-place
- stable sort
- iterative
- recursive
- mutable and immutable
- overflow
- underflow
- collision in hashing

## Practical Rule

Always compare the solution complexity with the constraints before coding.
