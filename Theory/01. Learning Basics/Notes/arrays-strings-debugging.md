# Arrays, Strings, and Debugging

Arrays and strings are the first place where indexing mistakes appear.

## Arrays

- store same-type elements
- support indexed access
- usually zero-based

## Strings

- sequence of characters
- often treated like arrays of characters
- common operations: traversal, comparison, substring, reverse

## Debugging Skills

- print intermediate values
- dry run on paper
- test smallest and largest cases
- check index boundaries
- verify assumptions after every loop

## Frequent Mistakes

- accessing index `n` in an array of size `n`
- forgetting to reset counters
- mixing character and integer logic
