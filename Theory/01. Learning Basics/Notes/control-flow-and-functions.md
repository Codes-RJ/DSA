# Control Flow and Functions

This note covers the logic-building blocks used in every DSA problem.

## Conditions

Use conditions to choose between branches of execution.

- `if` checks one condition
- `if-else` handles two outcomes
- `else-if` handles multiple cases
- `switch` is useful when matching fixed values

## Loops

Loops repeat work until a condition changes.

- `for` loop: best when the number of iterations is known
- `while` loop: best when repetition depends on a condition
- `do-while` loop: runs at least once

## Functions

Functions break large logic into reusable pieces.

### Why functions matter in DSA

- improve readability
- reduce repetition
- make recursion possible
- allow helper methods for subproblems

## Common Beginner Errors

- forgetting to update loop variables
- using wrong loop boundaries
- returning too early
- not handling empty input
