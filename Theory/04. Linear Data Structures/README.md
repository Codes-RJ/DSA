# Linear Data Structures

Linear structures store elements in a sequence.

## Arrays

- fixed or dynamic sequential storage
- fast random access
- useful for indexing, prefix sums, binary search

## Strings

- sequence of characters
- common in parsing, matching, hashing, and dynamic programming

## Linked Lists

- singly linked list
- doubly linked list
- circular linked list
- useful when insertion and deletion matter more than indexed access

## Stack

- last in, first out
- used in parsing, undo logic, recursion simulation, monotonic stack

## Queue

- first in, first out
- useful in BFS, scheduling, buffering

## Deque

- insert and delete from both ends
- useful in sliding window maximum and custom queue logic

## Hashing

- map keys to values
- average constant-time insert, search, delete
- watch for collisions

## What to Learn for Each Structure

- representation
- core operations
- complexity
- advantages and disadvantages
- common interview questions
