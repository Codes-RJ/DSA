# Greedy, Dynamic Programming, and Graph Algorithms

## Greedy

Greedy chooses the locally best option at each step.

Use it only when the choice can be proven safe.

## Dynamic Programming

DP is useful when:

- subproblems overlap
- the optimal answer can be built from smaller optimal answers

### Standard DP workflow

1. define state
2. write transition
3. choose memoization or tabulation
4. optimize space if possible

## Graph Algorithms

- BFS for shortest path in unweighted graphs
- DFS for connectivity and structure
- Dijkstra for positive weighted shortest path
- Kruskal and Prim for minimum spanning tree
